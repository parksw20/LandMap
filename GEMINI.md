# God Slayer: Abyss - 부동산 실거래가 시각화 시스템 (GEMINI)

공공데이터 포털의 부동산 실거래가 데이터를 수집 및 가공하여 카카오맵 API 기반의 계층형 시각화와 정밀 분석을 제공하는 도구입니다.

## 🚀 핵심 기능 및 기술 사양

### 1. 계층형 시각화 (Hierarchical Visualization)
사용자의 줌 레벨에 따라 데이터의 밀도를 지능적으로 조절하여 가독성과 성능을 동시에 확보합니다.

*   **1단계: 시/도 (Zoom 9 이상)**
    *   표시: 지역명 / 대표 평형 및 평균가 (ex. 서울특별시 / 34평 15.5억)
*   **2단계: 구/시 (Zoom 7 ~ 8)**
    *   표시: 구 명칭 / 대표 평형 및 평균가 (ex. 강남구 / 34평 22.3억)
*   **3단계: 법정동 (Zoom 5 ~ 6)**
    *   표시: 동 명칭 / 대표 평형 및 평균가 (ex. 개포동 / 34평 20.1억)
*   **4단계: 단지/건물 (Zoom 1 ~ 4)**
    *   표시: 대표 평형 / 평균가 (ex. 34평 / 18.5억)
    *   상호작용: 클릭 시 측면 패널에 상세 거래 리스트 출력

### 2. 정밀 데이터 분석 규칙

#### 💰 금액 표기 규칙
*   **1억 원 이상**: 억 단위 소수점 첫째 자리 반올림 표기 (ex. 7억 5555만 → **7.6억**)
*   **1억 원 미만**: 만 단위 콤마 표기 (ex. 1,555만 → **1,555만**)

#### 📊 대표 데이터 산출 (Marker Info)
*   해당 지역/단지에서 가장 거래 건수가 많은 **최빈 평형(Mode Area)**을 대표 평형으로 선정
*   해당 평형의 거래 금액들에 대한 **평균치**를 대표 가격으로 산출

#### 🔝 상세 리스트 정렬 우선순위
1.  **거래 유형**: 매매 > 전세 > 월세 순
2.  **계약일**: 최신순 (내림차순)
3.  **거래가**: 가격 낮은순 (오름차순)
4.  **층수**: 높은 층순 (내림차순)

## 📁 데이터 아키텍처 (Data Sharding)

대용량 실거래 데이터(월 수십 MB)를 브라우저에서 효율적으로 처리하기 위한 분산 저장 전략을 사용합니다.

*   **경로 구조**: `data/hierarchy/{YYYYMM}/{Type}/`
*   **요약 데이터**: `summary_*.json` (시도/구군/동 단위 집계 정보, 초기 로딩용)
*   **상세 데이터 (Shards)**: `details/{Sido}_{Gungu}.json`
    *   단지별 상세 거래 내역(계약일, 동, 층, 대지면적, 임차기간, 갱신여부, 종전가격 등) 포함
    *   **Multi-Shard Loading**: 지도의 가시 영역(Bounds) 내에 포함된 모든 구의 샤드 파일을 비동기로 병합 로드하여 경계 지역 데이터 누락 방지

## 🛠 시스템 구성 (Python Modules)

각 모듈은 독립적인 역할을 수행하며, `data_manager.py`를 통해 통합 제어됩니다.

1.  **`data_manager.py` (Core Orchestrator)**
    *   **역할**: 시스템 엔트리 포인트 및 전체 작업 흐름 제어.
    *   **기능**: `data/YYYY` 폴더 내의 엑셀 파일을 탐색하여 파싱, 지오코딩, 계층형 데이터 빌드 과정을 순차적으로 실행. 작업 완료 후 `manifest.json`을 갱신하여 웹 프론트엔드와 동기화.
2.  **`land.py` (Data Collector)**
    *   **역할**: 공공데이터 포털 API를 통한 실거래 데이터 수집.
    *   **기능**: 국토교통부 실거래가 API(아파트, 연립다세대, 단독다가구)를 호출하여 엑셀 파일로 저장. `LAWD_서울_경기.csv`에 정의된 지역 코드를 기반으로 대량 수집 수행.
3.  **`geo_cache.py` (Geocoding & Cache Manager)**
    *   **역할**: 주소의 위경도 좌표 변환 및 `address_cache.json` 관리.
    *   **기능**: 1차적으로 로컬 캐시에서 좌표를 검색하며, 없을 경우 **카카오 로컬 API를 실시간 호출**하여 좌표를 획득. 획득된 좌표는 캐시에 영구 저장되어 API 호출 횟수를 최소화함.
4.  **`excel_parser.py` (Data Normalizer)**
    *   **역할**: 다양한 양식의 엑셀 데이터를 표준 규격으로 정규화.
    *   **기능**: 수집된 엑셀 시트를 분석하여 매매/전월세를 구분하고, 컬럼명을 매핑함. 특히 **지오코딩 정확도를 위해 `주소` 컬럼을 최우선으로 활용**하도록 설계됨.
5.  **`hierarchy_builder.py` (Statistics & JSON Builder)**
    *   **역할**: 통계 산출 및 시각화용 JSON 샤드 생성.
    *   **기능**: 수집된 데이터를 시도/구군/동 단위로 그룹화하여 거래 건수, 가격 범위, 대표 평형 등을 계산. 웹에서 즉시 사용 가능한 커스텀 인덴트 포맷의 JSON 생성.
6.  **`fix_cache.py` (Maintenance Tool)**
    *   **역할**: 캐시 데이터의 정합성 유지 및 보정.
    *   **기능**: 캐시에 저장된 좌표의 순서([경도, 위도] vs [위도, 경도])를 검증하고 잘못된 데이터 형식을 일괄 수정.
7.  **`inspect_excel.py` / `check_cache_keys.py` (Utility)**
    *   **역할**: 데이터 수집/가공 전후의 상태 점검.
    *   **기능**: 엑셀의 컬럼 구조를 미리 확인하거나, 캐시에 등록된 주소 키의 형식을 샘플링하여 출력.

### 🤖 사용자 인터페이스 (UI/UX)

*   **마우스 오버 툴팁 (Context Window)**
    *   탭(Tab) 형식 그리드 레이아웃: `유형 | 건수 | 가격대` 정렬
    *   색상 강조: 매매(빨강), 전세(파랑), 월세(초록) 라인 전체 테마색 적용
    *   플리커링 방지: `pointer-events: none` 및 `itemKey` 상태 체크를 통한 안정적 표시
*   **다중 필터링 시스템**
    *   **기간 선택**: 최대 3개월까지 체크박스 다중 선택 및 데이터 실시간 병합
    *   **주택 유형**: 아파트(파랑), 빌라(녹색), 주택(빨강), 오피스텔(하늘색) 테마 적용
    *   **거래 유형**: 체크 해제 시 해당 거래가 없는 마커는 지도에서 즉시 제거
*   **실시간 검색**: 현재 로드된 전체 데이터를 대상으로 자동 완성 및 위치 이동 기능 제공

## ⚠️ 실행 가이드

1.  **공공 데이터 수집**: `python land.py -m 202604` 특정 월 기준으로 데이터를 쌓음 (현재 윈도우 `스케줄`로 돌아가고 있음)
2.  **데이터 업데이트**: `data/YYYY` 폴더에 엑셀 파일 확인 후 `python data_manager.py` 실행. (이 단계에서 지오코딩 및 JSON 빌드가 자동 진행됨)
3.  **웹 실행**: `실행.bat`을 실행하여 로컬 서버(Port 8000) 구동 후 브라우저 접속.
4.  **API 키 설정**: `data_manager.py`와 `land.py` 내의 카카오 API 키 및 공공데이터 API 키(Keyring 사용)를 확인하십시오.
